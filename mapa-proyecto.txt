────────────────────────────────────────────────────────────────────────────────
RAÍZ / CONFIG
────────────────────────────────────────────────────────────────────────────────

package.json
  Dependencias del proyecto: React 19, Firebase, Vite, Tailwind, Vertex AI SDK.
  Define scripts: dev, build, preview.

tsconfig.json
  Configuración TypeScript (target ES2022, JSX react-jsx, bundler resolution).

vite.config.ts
  Config de build y dev server con Vite para Vercel deployment.

tailwind.config.cjs
  Estilos Tailwind: componentes custom, colores brand, plugins.

postcss.config.cjs
  PostCSS + Autoprefixer para procesamiento de CSS.

vercel.json
  Config de deploy en Vercel: rutas, env vars, rewrites.

env.example
  Template de variables de entorno (GEMINI_API_KEY, Firebase keys, Upstash tokens).

index.html
  Entrypoint HTML que monta React en el div #root.

types.ts
  Tipos globales compartidos: ContentType, AvatarProfile, ProductProfile, GenerationSet, OutfitItem.
  Es la fuente de tipos core usada por todos los módulos.

firestore.rules
  Reglas de seguridad Firestore: lectura/escritura restringida por UID de usuario.

storage.rules
  Reglas de seguridad Cloud Storage: solo usuarios autenticados pueden subir.

metadata.json
  Metadata simple del proyecto (título, versión, descripción).

PRICING_BRIEF.md
  Especificación de planes y precios (Free/Explorer/Starter/Pro/Studio/Admin).

test-image.ts
  Script manual para testear generación de imágenes directamente contra la API.

SRC/SERVICES/ — Servicios core de negocio
────────────────────────────────────────────────────────────────────────────────

src/services/userService.ts
  CRUD completo de usuarios en Firestore: avatars, products, sets, stats, créditos.
  Fuente principal de acceso a datos del usuario.

src/services/firestoreUtils.ts
  Error handling y logging centralizado para todas las operaciones Firestore.

src/services/creditsService.ts
  Sistema de créditos: deduct, reset de período, getEffectiveCredits.
  Fuente única de verdad del saldo; usado por AuthContext y useCreditGuard.

  Config de costos: MODEL_CREDIT_COST (gemini=2cr, seedream=1cr), CREDIT_COSTS por acción.
  Cambiar aquí afecta el costo de TODAS las operaciones de crédito.

src/services/checkoutService.ts
  Integración con LemonSqueezy (payment gateway): crear sesión de checkout.

src/services/geminiService.ts
  Cliente central de Gemini: wrappers de imageApiService + content API.
  Expone extractAvatarProfile, analyzeProduct, analyzeOutfit, generateText.

src/services/imageApiService.ts
  Cliente universal para /api/gemini/image: startJob (→202) → polling cada 2s hasta 90 intentos.
  Callback onStatusChange para actualizar UI. Maneja retry silencioso.

src/services/avatarCloneService.ts
  Cliente para /api/avatar/clone: startClone → getCloneStatus → waitForCloneComplete.

src/services/cloneImageService.ts
  Clona/transforma una imagen individual llamando la API correspondiente.

src/services/ugcApiService.ts
  Llama el endpoint de generación UGC multi-shot.

src/services/dbService.ts
  Wrapper de compatibilidad sobre Firestore (getAvatars, saveAvatar, getSets).
  Redirige a userService; mantener por compatibilidad con módulos viejos.

src/services/generationHistoryService.ts
  Guarda y recupera historial de generaciones (shots, metadata, timestamps) en Firestore.

src/services/sceneAnalysisService.ts
  Analiza escenas para UGC: detecta fondo, iluminación, mood para directivas.

src/services/avatarService.ts
  Gestión de avatares guardados: get, save, delete desde Firestore.

src/services/missionsService.ts
  Sistema de misiones/quests (daily, weekly) con cooldown y recompensas en créditos.

src/services/referralService.ts
  Códigos referral: stats de uso, reembolso de créditos, códigos especiales.

src/services/promptRevealService.ts
  Revelar u ocultar el prompt de imágenes en la galería pública.

src/services/imageUtils.ts
  Comprimir imágenes, descargar individualmente, crear ZIP de múltiples imágenes.

API/ — Endpoints Vercel Serverless
────────────────────────────────────────────────────────────────────────────────

api/_middleware.ts
  Rate limiting (Redis Upstash), CORS, validación de auth, security headers.
  Funciones: checkRateLimit (30img/h, 120data/h), validateBase64Image, validatePrompt, setSecurityHeaders.

api/gemini/content.ts
  POST /api/gemini/content: extractAvatarProfile, analyzeProduct, analyzeOutfit, generateText, chat.
  Usa gemini-2.5-flash en us-central1 para análisis de contenido (texto/JSON).

api/gemini/image.ts
  POST /api/gemini/image: orquesta generación async → encola en QStash (devuelve jobId 202).
  GET /api/gemini/image?jobId=...: polling de estado del job hasta resultado/error.

api/gemini/image-worker.ts
  Worker QStash: recibe job, llama Gemini imagen, guarda resultado base64 en Redis.

api/gemini/seedream-worker.ts
  Worker QStash para modelo Seedream (alternativa rápida, 1cr vs 2cr de Gemini).

api/gemini/ugc-worker.ts
  Worker QStash para generación UGC multi-shot: orquesta batch de shots en paralelo.

api/avatar/clone.ts
  POST /api/avatar/clone: startClone → getJobStatus. Orquesta Worker + Firestore.

api/avatar/clone-worker.ts
  Worker async: llama Gemini para extractAvatarProfile, guarda resultado en Redis.

api/history.ts
  GET /api/history: recupera generationHistory del usuario desde Firestore.

api/webhooks/lemonsqueezy.ts
  Webhook LemonSqueezy: recibe notificaciones de pagos, valida firma HMAC, actualiza plan en Firestore.
